# 마이링크(My Link) 사용자 시나리오

본 문서는 마이링크 서비스에서 발생할 수 있는 모든 사용자의 행동 흐름을 "사용자는 ___하기 위해 ____한다" 형태의 시나리오로 정의합니다. 

---

## 1. 소유자 (Owner) 시나리오

### 1.1 회원가입 및 초기 설정
* **사용자는** 마이링크 서비스를 시작**하기 위해** 구글(Google) 계정 연동 버튼을 클릭하여 소셜 로그인**한다.**
* **시스템은** 사용자의 고유한 프로필 주소를 설정**하기 위해** 구글 지메일(Gmail) 계정의 `@` 앞부분을 추출하여 닉네임(`displayName`)으로 자동 등록**한다.**

### 1.2 프로필 관리 (인라인 편집)
* **사용자는** 자신의 프로필 닉네임을 변경**하기 위해** 닉네임(`displayName`) 텍스트 영역을 클릭하여 인라인으로 텍스트를 수정**한다.**
* **사용자는** 자신의 실제 이름(본명)을 명시**하기 위해** 실제 이름(`username`) 텍스트 영역을 클릭하여 인라인으로 텍스트를 수정**한다.**
* **사용자는** 방문자에게 자신을 간단히 소개**하기 위해** 한 줄 소개(Bio) 텍스트 영역을 클릭하여 인라인으로 내용을 작성**한다.**
* **사용자는** 인라인으로 수정한 텍스트를 최종 저장**하기 위해** 엔터(Enter) 키를 누르거나 마우스로 화면의 다른 곳을 클릭(Blur) **한다.**

### 1.3 링크 관리 (추가, 수정, 삭제)
* **사용자는** 새로운 외부 사이트를 프로필에 등록**하기 위해** '링크 추가' 버튼을 클릭**한다.**
* **사용자는** 추가된 링크 블록에 실제 접속 주소를 설정**하기 위해** URL 텍스트 영역을 클릭하여 인라인으로 주소를 입력**한다.**
* **사용자는** 방문자가 링크의 목적을 쉽게 알게 **하기 위해** 링크 타이틀 텍스트 영역을 클릭하여 인라인으로 제목을 수정**한다.**
* **시스템은** 링크 아이템을 시각적으로 돋보이게 **하기 위해** 사용자가 입력한 URL을 바탕으로 구글 파비콘 API를 호출하여 아이콘을 자동 적용**한다.**
* **시스템은** 파비콘 로딩이 불가능한 예외 상황(잘못된 URL, 파비콘 부재 등)을 매끄럽게 처리**하기 위해** 기본(Default) 아이콘을 대체하여 렌더링**한다.**
* **사용자는** 더 이상 사용하지 않거나 잘못 등록된 링크를 프로필에서 지우기 **위해** 해당 링크 아이템 우측의 '삭제' 버튼(또는 아이콘)을 클릭**한다.**

---

## 2. 방문자 (Visitor) 시나리오

### 2.1 페이지 방문 및 탐색
* **사용자는** 소유자의 다양한 활동(SNS, 포트폴리오 등)을 확인**하기 위해** 공유받은 마이링크 프로필 URL(`도메인.com/닉네임`)을 클릭하여 퍼블릭 페이지에 접속**한다.**
* **사용자는** 소유자의 기본 정보(실제 이름, 닉네임, 한 줄 소개)를 확인**하기 위해** 화면 상단의 프로필 영역을 열람**한다.**

### 2.2 링크 클릭 및 이동
* **사용자는** 소유자의 특정 외부 서비스(예: 인스타그램, 깃허브)로 이동**하기 위해** 리스트에 나열된 링크 아이템 중 하나를 클릭**한다.**
* **시스템은** 어떤 링크가 인기가 많은지 성과를 측정**하기 위해** 방문자가 링크를 클릭하는 순간 해당 링크의 클릭 조회수(Click Count)를 +1 증가하여 데이터베이스에 기록**한다.**
